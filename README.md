# APPLE 序號查詢工具

本專案為門市人員設計，是一套獨立的靜態網頁查詢工具（單一 HTML 檔案）。讓門市人員能夠透過電腦掃描槍或行動裝置相機，快速掃描 Apple 產品外盒的二維條碼（GS1 Digital Link 格式），並自動解析出 S/N 序號、IMEI 1、IMEI 2、EID 及商品料號等重要條碼資訊。

解析後的序號會在網頁上顯示為一維條碼（CODE128），門市人員可使用掃描槍直接讀取螢幕上的條碼帶入 POS 系統，或使用一鍵複製功能。

## 核心功能

1. **兩種輸入方式**：
   - **掃描器/手動輸入**：將游標聚焦於輸入框，使用實體掃描器掃描外盒 QR Code 後，系統偵測到 `Enter` 鍵會自動觸發解析。
   - **鏡頭掃描**：支援行動裝置開啟網頁後，直接調用後置鏡頭，提供十字對焦線與自動對焦。
2. **GS1 標準解析**：
   - 解析 UPC/GTIN (AI 01)、S/N 序號 (AI 21)、主要 IMEI (AI 8040)、次要 IMEI 2 (AI 8041) 以及 eSIM EID (AI 8042)、商品料號 (AI 240)、藍牙/WiFi ID 等。
   - 針對 Mac 等設備可能出現之 S/N 尾端黏接 Part Number 的特殊 GS1 Digital Link 邊界狀況進行了自動相容截斷處理。
3. **一維條碼生成**：
   - 自動將核心序號（S/N, IMEI, EID）轉化為 CODE128 一維條碼，條碼背景設為**純白色**，確保各種實體紅光與影像掃描器皆可流暢掃描。
4. **極簡 Apple 官網風 UI**：
   - 採用 Apple 官方淺灰與純白配色、高清晰度字體階層與優雅的圓角按鈕設計。

## 部署與使用方法

### 1. 本機使用
- 下載或複製本專案的 `index.html`，在電腦上雙擊開啟即可使用。

### 2. GitHub Pages 部署 (推薦)
由於瀏覽器安全機制，**鏡頭掃描功能必須在安全的 HTTPS 環境下才能正常運作**（本機測試使用 `localhost` 除外）。部署於 GitHub Pages 能免費獲得自動 HTTPS。

**部署步驟**：
1. 將本專案推送至您的 GitHub 儲存庫。
2. 前往儲存庫的 **Settings** -> **Pages**。
3. 在 **Build and deployment** 下的 **Branch**，選擇 `main` (或您上傳的分支)，並點選 **Save**。
4. 稍等數分鐘，GitHub 就會為您生成一個 HTTPS 的網頁連結！

## 資料安全
本工具為**純前端靜態網頁**，所有的條碼解析、條碼生成及資料複製程序均完全在本機瀏覽器中完成，**不會傳送或上傳任何資訊至任何外部伺服器**，保障門市營運資料的絕對隱私與安全。
